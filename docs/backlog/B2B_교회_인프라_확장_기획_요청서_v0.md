## 사용자 작성 프롬프트 원문 (그대로 보존)

# 묵상 앱 → 교회 인프라 확장 전략 & B2B 비즈니스 모델 설계 요청

## 역할
당신은 SaaS B2B 전환 전략, 교회·종교기관 IT 도입 컨설팅, 그리고 구독 모델 설계에 깊은 경험을 가진 **프로덕트 전략 컨설턴트**다. 린 스타트업 방법론과 기독교 교회 운영 실무를 동시에 이해하고 있다.

## 배경

### 앱의 현재 상태
- "묵상에서 시작하는 삶 운영 시스템" 앱이 이미 완성되어 있음.
- 핵심 모듈(나무): 묵상 → 목표 타임박싱 → 캘린더 연동, 도트 콜렉터, AI 분석 리포트, 워크플로우, 인물·조직 평가, 원칙·판례 기반 의사결정 보조 AI, 가계부 도트, 온보딩·튜토리얼 미션.
- 1차 클로즈드 베타 (10명, 2주, 개인 묵상 중심) 진행 예정.
- 2차 오픈 베타에서 공동체 기능 추가 후 교회 단위 검증 예정.

### 확장 기획 (숲)
기존 모듈을 재조합하여 교회 인프라로 확장. 기획 중인 기능:
1. **예배 준비 행정 기능** – 설교 본문 연동, 순서지 작성, 봉사자 배정, 공지 전달, 회의록 정리 등 매주 반복 워크플로우.
2. **찬양 스코어 제작** – 악보 생성·편집·공유, 찬양팀 연습 관리.
3. **대·중·소그룹 관리** – 그룹 생성, 멤버 관리, 출석·참여 추적, 그룹 리더 대시보드.
4. **디자인 에디터** (검토 중) – 순서지·홍보물·공지 디자인 인앱 제작. 개발 비용 대비 효용 미확정, 초기에는 외부 연동(Canva, 미리캔버스)으로 대체 가능.
5. **"예배 → 삶 적용" 루프** – 예배/설교에서 받은 것을 개인 묵상·목표·타임박싱으로 연결하는 흐름. 앱의 원래 DNA("묵상 → 실행")를 교회 단위로 스케일한 것.

### 시장 데이터
- 글로벌 영적 웰니스 앱: 2024 $2.16B → 2033 $7.31B (CAGR 14.63%)
- 한국 기독교 인구 ~1,600만, 교회 약 6만 개
- 해외 한인교회 5,100~5,300개 (미국 3,514개)
- 재외동포 약 700만 명 (파리 2.5만, 홍콩 1.8만, 일본 96만)
- 경쟁사 Hallow: 교구(parish) B2B 전략으로 $157M 펀딩, ARR $60M

### 구독 모델 기본 가정
- 개인 B2C: 월 4,900~6,900원
- 그룹 리더: 월 9,900원 (멤버 10명 포함)
- 교회 B2B: 월 20만~100만 원 (규모별)
- 기업·기관 B2B: 월 50만~200만 원
- 앱스토어 수수료: 연 $1M 이하 15%, 초과 30%

### 수익 규모 레퍼런스
| 구독자 수 | 월 총매출 | 연 세후 순이익 (추정) |
|---|---|---|
| 1,000명 | 590만 원 | 4,000~4,500만 원 |
| 10,000명 | 5,900만 원 | 4.5~5억 원 |
| 100,000명 | 5.9억 원 | 24~34억 원 |
| 1,000,000명 | 59억 원 | 210~300억 원 |
* 위 수치는 개인 B2C 기준. B2B가 추가되면 객단가·잔존율 모두 상승.

## 산출물 1: B2C → B2B 전환 전략서

### 포함 항목
- **전환 논리**: 왜 개인 묵상 앱이 교회 인프라가 될 수 있는가. 기존 모듈이 교회 맥락에서 어떻게 재조합되는지 매핑.
- **B2B 가치 제안**: 교회 의사결정자(담임목사, 부목사, 행정간사, 교육부서장)별로 각각 어떤 가치를 느끼는가.
- **진입 전략**: 첫 교회 고객을 어떻게 확보하는가. 오픈 베타 참여자 중 소그룹 리더 → 교회 전체 도입으로 이어지는 경로 설계.
- **Hallow 교구 전략 벤치마킹**: Hallow의 parish partnership 모델을 분석하고, 한국 교회 구조(장로교·감리교·침례교 등 교단별 의사결정 방식)에 맞게 변형한 전략 제시.
- **잠금 효과(Lock-in) 분석**: 예배 준비 워크플로우, 데이터 축적, 그룹 관계망 등 각 기능이 만드는 전환 비용을 정량적으로 설명.

---

## 산출물 2: 확장 기능 우선순위 & 로드맵

### 포함 항목
- 아래 5개 기능에 대해 **개발 난이도(상/중/하) × B2B 전환 기여도(상/중/하) × "묵상→실행" 서사 연결 강도(상/중/하)** 매트릭스 작성:
  1. 예배 준비 행정
  2. 찬양 스코어 제작
  3. 대·중·소그룹 관리
  4. 디자인 에디터
  5. "예배 → 삶 적용" 루프
- 매트릭스 결과에 따른 **출시 순서 권장안** 및 각 기능의 **MVP 범위** 정의.
- 디자인 에디터에 대한 **Go / 외부 연동 / 보류** 판단 기준 (어떤 시그널이 나오면 직접 개발할 것인가).
- 오픈 베타 → 정식 출시 → 6개월 → 1년 → 2년 타임라인에 기능 배치.

## 산출물 3: B2B 가격 모델 설계

### 포함 항목
- **교회 규모별 티어 설계**: 소형(50인 이하) / 중형(50~200인) / 대형(200~1,000인) / 메가(1,000인+) 교회 각각에 대해 월 구독료, 포함 기능, 추가 옵션 정의.
- **기업·기관 티어**: 미션 스쿨, 신학대학원, 기독교 기업, 선교단체 등 대상별 가격·기능 차등.
- **개인 → 그룹 → 교회 업셀 경로**: 개인 사용자가 그룹 리더로, 그룹 리더가 교회 도입을 요청하는 자연스러운 업셀 퍼널 설계.
- **무료 티어 전략**: 교회에 무료 체험을 어떤 범위까지 허용할 것인가. 무료 → 유료 전환 트리거.
- **연간 결제 할인, 교단 단위 볼륨 계약** 등 고급 가격 전략.

## 산출물 4: 오픈 베타 공동체 기능 검증 설계

### 검증해야 할 핵심 가설
- **가설 A**: 소그룹 리더가 예배 준비 워크플로우를 매주 실제로 사용한다.
- **가설 B**: "예배 → 삶 적용" 루프를 통해 개인 묵상 습관이 유지 또는 강화된다 (행정 도구로만 인식되지 않는다).
- **가설 C**: 그룹 관리 기능 사용 후 리더가 "교회 전체 도입"을 자발적으로 요청한다.
- **가설 D**: 공동체 기능이 개인 사용자의 이탈률을 낮춘다.

### 포함 항목
- 각 가설별 **측정 지표, 성공 기준, 데이터 소스(GA4 이벤트 / SWAN 로그 / 설문 / 인터뷰)**.
- 오픈 베타 **참여자 구성 권장안**: 개인 사용자 몇 명, 소그룹 리더 몇 명, 예배 담당자 몇 명.
- 오픈 베타 **기간 및 주차별 체크포인트**.
- 가설 B가 실패할 경우(묵상과 행정이 분리될 경우)의 **피벗 시나리오**: 묵상 앱으로 유지 vs 교회 행정 SaaS로 전환 vs 양쪽 분리 운영.

---

## 산출물 5: 경쟁 포지셔닝 맵

### 포함 항목
- **2축 포지셔닝 맵**: X축 = 개인 영성 ↔ 교회 운영, Y축 = 콘텐츠 중심 ↔ 실행/생산성 중심. 기존 경쟁 앱(Hallow, Glorify, YouVersion, 생명의삶, 데일리성경, ChurchTools, Planning Center, Elvanto 등)과 본 앱의 위치 배치.
- **교회 운영 SaaS 경쟁사 분석**: Planning Center, ChurchTools, Elvanto, Tithe.ly, Subsplash 등 해외 교회 관리 플랫폼의 기능·가격·시장 점유율 요약. 이들과 본 앱의 차별점("묵상에서 시작"이라는 진입 경로).
- **한국 교회 행정 도구 현황**: 현재 사용 도구(카톡, 구글 시트, 한글, 교적 관리 프로그램 등)와 페인포인트 정리.
- **"묵상 → 교회 인프라" 포지션이 독점 가능한 이유**: 양쪽(영성 앱 / 교회 SaaS) 어디에서도 접근하지 않는 빈 공간인 근거.

## 산출물 6: 이탈률(Churn) & 잠금 효과(Lock-in) 시뮬레이션

### 포함 항목
- **B2C vs B2B 이탈률 시나리오**: 개인 월 이탈률 5% / 3% / 1%일 때 12개월 후 잔존 사용자 수 비교. 교회 B2B 연 이탈률 10% / 5% / 2%일 때 같은 비교.
- **잠금 레이어별 효과**:
  - Layer 1: 습관 잠금 (매일 묵상 루틴)
  - Layer 2: 데이터 잠금 (도트 콜렉터 축적 데이터)
  - Layer 3: 워크플로우 잠금 (예배 준비 프로세스 의존)
  - Layer 4: 관계망 잠금 (그룹·교회 네트워크)
  - Layer 5: 재무 잠금 (가계부 도트 히스토리)
  각 레이어가 추가될 때 이탈률이 어떻게 변하는지 추정.
- **복합 시나리오**: B2C 1,000명 + 교회 20곳(교회당 평균 100명) 조합일 때 월 매출·연 매출·세후 이익 시뮬레이션.
- **"루틴이 된 앱은 해지가 아니라 삶을 바꾸는 것"**이라는 관점에서, 전환 비용이 금전적이 아닌 심리적·영적·관계적임을 강조.

---

## 산출물 7: 디자인 에디터 Go/No-Go 판단 프레임워크

### 포함 항목
- **Go 시그널**: 오픈 베타 또는 정식 운영 중 어떤 데이터가 나오면 직접 개발에 착수할 것인가 (예: CS 요청 빈도, 외부 도구 이탈률, B2B 교회의 명시적 요청 비율 등).
- **No-Go / 대체 전략**: Canva API 연동, 미리캔버스 링크, 템플릿 제공 등 경량 대안의 구현 방법과 한계.
- **개발 비용 대비 수익 추정**: 에디터 개발에 예상되는 시간·비용 vs 이 기능으로 인한 추가 구독 전환·이탈 방지 효과.
- **경쟁 리스크**: Canva가 교회 템플릿을 강화하거나, 교회 SaaS가 디자인 기능을 내장할 가능성과 대응.

## 전체 문서 기조
- **"나무는 완성되어 있고, 숲을 그리는 것이다"** – 신규 개발이 아닌 기존 모듈의 재조합임을 일관되게 반영.
- **"묵상 → 실행"이라는 DNA가 모든 확장의 기준** – 이 서사에서 벗어나는 기능은 후순위이거나 외부 연동으로 대체.
- **실무에서 바로 참고할 수 있는 구체적 수준** – 숫자, 기준, 조건이 있는 곳은 반드시 수치로 명시.
- 분량이 길면 산출물별로 나눠서 출력하고, "이어서 작성해줘" 요청 시 계속 이어갈 것.
