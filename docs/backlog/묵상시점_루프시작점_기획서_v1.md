## 1. 배경

### 1.1 현재 자리
- 묵상 = 어느 시점이든 가능 (새벽·아침·점심·오후·저녁·심야)
- 그런데 시스템은 자정 기준 단일 "오늘" 결만 인식
- **결과**: 시점에 따른 다음 단계로 흐르는 회로 자체가 없음

### 1.2 사용자 통찰 (2026-05-21)
> "아침에 하는 사람은 당일 계획을 하는 사람일거고, 저녁에 하는 사람은 다음날 계획을 하는 사람일거란말이지... 사용자가 이용하는 시간에 맞춰서 다음 순간을 계획하게 만들어줘야 할 것 같은데, 지금 베타 버전에는 그런 기능 자체가 없지 않나 싶어서."

### 1.3 핵심 통찰
**"자기 리듬을 시스템이 입력 없이 알아채는 결"** — 사용자가 수면 시간을 따로 입력하지 않아도, 묵상 시점·도트 시간으로 자기 결로 추측. Sanctum OS 정체성과 정합 가장 큼.

## 2. R1~R6 합의 묶음

### R1 — 기본 결
- 사용자 입력 X (수면 시간대 안 물어봄)
- 시간대 휴리스틱 디폴트 (1차) → 학습 (2차)
- 매번 작은 토글로 사용자 갈아끼움
- 묵상 시간 정보 저장 (학습 데이터)
- "오늘 / 내일" 어휘 유지

### R2 — 시점 buckets

| 시간대 | 결 | 디폴트 |
|---|---|---|
| 5~12시 | 오늘 시작 | 오늘 |
| 12~17시 | 한참 진행 중 | 오늘 + 내일 둘 다 노출 |
| 17~24시 | 자기 전 결 | 내일 |
| 0~5시 | 심야 | 오늘 |

### R3 — UI 자리
- 토글 자리 = 묵상 끝 자리 "지금부터 계획하기" CTA + [오늘] [내일]
- 묵상 안 한 사용자 = 진입 시점 휴리스틱 그대로
- ✨ **NEW**: 묵상 시점이 시간표에 자동 도트로 자리잡힘

### R4 — 미션 카탈로그
- `any_meditation` 신규 메인 미션 (어느 시점이든 묵상 1회)
- `morning_meditation`·`evening_meditation` 시점 보너스 유지
- `plan_first_toggle` 신규 미션: "오늘·내일 계획 한 번 짜기"

### R5 — 데이터 마이그레이션
- 옛 묵상 노트 시간 정보 백필 X (그대로 두기)
- 옛 묵상 도트 시간표 노출 X (그대로 안 보이게)
- 기존 사용자에게 1회 안내 모달

### R6 — 1차 vs 2차
- 1차 = 자리 ①~⑧ 모두 자리잡힘
- 2차 = 학습 알고리즘 (수면 시간대 자동 추측)

## 3. 1차 베타 구현 자리 — 8건

### 자리 ① — 묵상 도큐먼트 시간 필드
- **파일**: src/ui/todayView.js (saveMeditationDoc), src/config/encryptionPolicy.js
- **신규 필드**: `meditationTime: 'HH:MM'` (평문 메타 — 인덱싱·정렬·통계용)
- **계산**: `new Date()` 시점에서 hour·minute 추출
- **encryptionPolicy.js 정합**: meditations 평문 메타에 추가 (본문·기도는 그대로 암호화)

### 자리 ② — 묵상 시점 시간표 자동 도트
- **파일**: src/ui/todayView.js (saveMeditationDoc 안 자동 도트 자리)
- **현재 결**: `kind='meditation'`, `timeSlot=null` → 시간표에 안 보임
- **갈음**: `timeSlot` 자동 계산 (현재 시점 → 15분 단위 slot 인덱스), `durationSlots = 4` (1시간 디폴트)
- **라벨**: "📖 묵상" + 본문 reference (예: "📖 묵상 — 시편 23편")

### 자리 ③ — 시점 휴리스틱 헬퍼 모듈
- **신규 파일**: src/ui/loopStartPoint.js
- **export 함수**:
  ```js
  getLoopStartHint(hour)
    → { defaultTarget: 'today' | 'tomorrow' | 'both', bucket: 'morning'|'noon'|'evening'|'midnight' }
  resolveTargetDate(target, baseDate)
    → 'YYYY-MM-DD' (오늘 → baseDate, 내일 → baseDate + 1)
  ```

### 자리 ④ — 묵상 끝 [오늘] [내일] 토글 CTA 카드
- **파일**: src/ui/todayView.js
- **위치**: 묵상 노트 저장 직후 자연 노출 카드 (묵상 섹션 아래)
- **결**:
  - 헤더: "지금부터 계획하기"
  - 토글: [🌤 오늘] [🌅 내일] (디폴트 = `getLoopStartHint` 결과)
  - CTA 버튼: "계획 자리로 →"
- **클릭 시**: 결단·시간표 자리로 자연 진입 (해당 날짜 결로)

### 자리 ⑤ — 12~17 두 칸 동시 노출
- **파일**: src/ui/todayView.js
- **결**: `getLoopStartHint` 결과가 `'both'`이면 두 카드 동시 노출
  ```
  ┌─ 🌤 오늘 남은 시간 ─┐  ┌─ 🌅 내일 ──┐
  │ → 결단·시간표 자리  │  │ → 결단·시간표 자리│
  └────────────────────┘  └────────────────┘
  ```
- 그 외 시점은 단일 카드 + 위 작은 토글로 갈아끼움

### 자리 ⑥ — 진입 시점 휴리스틱 (묵상 안 한 결)
- **파일**: src/ui/todayView.js (또는 dashboard.js)
- **결**: 묵상 안 한 사용자가 화면 들어올 때 현재 시간 보고 같은 휴리스틱 적용
- 화면 첫 진입 자리에 같은 카드 노출

### 자리 ⑦ — 미션 카탈로그 갈아끼움
- **파일**: src/config/missionCatalog.js
- **신규 미션 2건**:
  - `any_meditation` (메인): 어느 시점이든 묵상 1회
  - `plan_first_toggle`: "오늘·내일 계획 한 번 짜기" — 묵상 끝 토글 1회 클릭으로 트리거
- **기존 자리**: morning·evening 시점 보너스 그대로 유지

### 자리 ⑧ — 1회 안내 모달
- **파일**: src/ui/loopStartPoint.js (또는 todayView.js)
- **결**: 다음 묵상 끝 자리 1회 노출 — "계획 자리가 자리잡혔어요" + 새 결 설명
- **플래그**: `localStorage['sanctum.loopStartPoint.intro.v1.seen']`로 한 번만 자리잡힘

## 4. 2차 베타 결 — 학습

### 학습 입력
- 누적 도트 시간 분포
- 묵상 시간 분포 (자리 ① 결로 누적)
- 사용자 토글 결 (디폴트와 다르게 갈아끼움 시점·빈도)

### 학습 출력
- 수면 시간대 자동 추측 (예: 23~07시)
- 디폴트 정확도 자기 결로 정밀해짐
- "보니까 보통 23시쯤 마지막 활동이세요. 내일 계획 자리로 둘게요" 자연 안내

## 5. 미해결 자리 (구현 자리에서 결로 자리잡힘)

- 묵상 도트 라벨 — "📖 묵상" 단순 vs "📖 묵상 — 시편 23편" reference 포함

- 안내 모달 카피 정확한 결
