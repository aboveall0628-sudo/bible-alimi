## §1. 정체성 검증 가설 (베타의 본질)

> **"거대한 유기체가 한 흐름으로 이해되는가"**

Sanctum OS 정체성:
- "걸어다니는 성경" — 묵상·기도가 모든 회복의 종착지
- 원칙·판례·목표·도트·리포트가 **한 흐름의 인과 체인** (메모리 [feedback_organism_architecture.md](../../.claude/projects/c--Users-MSI-Desktop----Sanctum-OS/memory/feedback_organism_architecture.md))

**1차 베타의 본질**: 베타 테스터가 이 시스템을 사용한 후 **"이거 다 연결돼 있구나"** 가 보이는가, 아니면 **"이거 기도 일기 앱 / 도트 트래커 / 평가 도구 따로따로네"** 로 보이는가의 분기.

→ 1차는 **정성** 1순위, **정량** 보조.
→ 정량 보조 신호: GA4 모듈간 이동률 (도트 → 원칙 → 목표 / 분별의 자리 → 폐기 게이트 / 묵상 → 기도 → 리포트), 인앱 피드백 누적량.

**2차 베타의 본질**: 공동체 기능이 같은 흐름에 자연스럽게 합류하는가. 1차의 개인 흐름이 관계 신뢰도·양치기 소년·기여도로 확장될 때도 "한 유기체" 감각이 유지되는가.

## §3. 1차 → 2차 게이트

### §3.1 게이트 정의

> **1차 리더·전문가 전원의 "내 사람에게 권하겠음" 명시**

- "전원" = 리더 3 + 전문가 2 = **5명 모두**.
- 한 명이라도 부정·유보면 진입 보류 → 1차 데이터 보강 후 재게이트.

### §3.2 명시 형식

> **서면 동의서** (인터뷰 후 별도 서명)

- 1:1 인터뷰 끝에 구두 의사 확인.
- 이후 짧은 서면 동의서 발송 → 서명 회수 (전자 서명 or PDF·종이).
- 동의서 항목 예: "본 시스템을 내 교인·소그룹·후배에게 권하는 데 동의합니다 / 단서 있음(서술) / 권하지 않겠습니다."

### §3.3 게이트 통과 후 별도 트랙들 (1↔2 사이의 큰 갭)

| 트랙 | 분량 |
|---|---|
| 🔐 **보안 재기획** (Q29 결정 — 2차 직전 진입) | 6~8일 |
| 🏠 **공동체 모듈 전체 통합 기획서** (신규 작성 필요) | 다수 세션 |
| 💰 **결제 모델 트랙** (별도, 결제·구독·후원형 분기) | 다수 세션 |
| 🧹 1차 데이터·인터뷰 회고 + 시스템 보강 | 1~2 세션 |
| 📣 마케팅 트랙 본격화 (시점은 §8에서 결정) | 별도 |

- A) 계속 일반 사용자 + 자기 교인 사용 동행 (맨토 윈테기)
- B) 공식 멘토 역할 (1차 수료 후 2차 새 유저 멘토링)
- C) 이탈 허용 — 표현했지만 결정은 그들에게
- (결합 가능)

### §4.4 2차 종료 기준

> **결정 유보** — 1차 데이터 본 후 설계.

후보 갈래 (§8):
- 개인별 도달 (1차와 동일 패턴)
- 공동체 단위 (소그룹별 N주 운영 후)
- 고정 기간 (예: 8주)

## §5. 혜택 설계

### §5.1 결제 무관 (지금 확정 가능)

**1차 (개척자)**:
- ✅ **영구 닉네임** (개인 정한 닉네임 영구 보존, 변경 불가 또는 1회 변경 정도)
- ✅ **"개척자" 칭호** (프로필 배지)
- ✅ **"Day 1 사용자" 마일스톤 표기** (가입 시점이 누적될수록 가치 ↑)

**2차**:
- 별도 칭호 (이름 미정 — 예: "선구자"·"동반자")
- 마일스톤 표기 (예: "베타 100인")

**버그헌터 보상**:
- 로드맵 #17 안에 별도 항목으로 이미 있음. 베타 혜택과 별개로 함께 운영.

### §5.2 결제 종속 (자리만, 채움 보류)

**결제 모델 트랙 결정 후 일괄 채움**:
- 평생 무료 / 등급 / 할인 / 후원자 권한 등 — 결제 모델(구독 vs 일회성 vs 후원 등) 결정 후 자리에 들어감.

### §5.3 1차 ↔ 2차 차등

- 1차 = 시스템 정체성 자체를 검증한 사람 → 가장 강한 명예
- 2차 = 공동체 흐름을 검증한 사람 → 차등 명예
- 가시적 표기 차이 + (결제 모델 결정 후) 권한·할인 차이

## §6. 정식 출시 + 1000명 마일스톤

### §6.1 정식 출시 시점

```
[2차 종료]
   │
   ├─ 결제 모델 트랙 완료
   ├─ 공동체 모듈 안정화
   └─ HC 대점검
   │
   ▼
[정식 출시]
   │
   ▼
[1000명 = 첫 마일스톤]
```

### §6.2 결제 모델 트랙 (별도)

**완전 미결** — 사용자 결정. 베타 기획서 안에서는 자리만 두고, 별도 트랙에서 진행:
- 구독 / 일회성 / 후원 / 무료+후원 분기 결정
- 결제 종속 혜택 채움 (§5.2)
- 1차·2차 참여자 영구 혜택 보장 구조 (예: 1차는 평생 무료 등급, 2차는 50% 할인 등 — 모델에 따라)

### §6.3 마케팅 트랙 (별도)

**본격화 시점 결정 유보** (§8).
- 베타 기획서에서는 "마케팅과 병행" 자리만 확보.
- 별도 트랙에서: 채널(교회 네트워크·SNS·유튜브·추천 코드), 메시지, 랜딩 페이지, 의가 등.

### §6.4 1000명 = 정식 출시 후 첫 마일스톤

- 베타 = 100명 안에서 종결
- 1000명은 마케팅 본격화 + 정식 출시 후 도달할 첫 마일스톤 지표
- 이 시점 = 시스템 안정성·결제 모델 작동·CS AI 작동 검증의 자연 트리거
