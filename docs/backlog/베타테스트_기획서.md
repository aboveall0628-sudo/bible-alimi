## §1. 정체성 검증 가설 (베타의 본질)

> **"거대한 유기체가 한 흐름으로 이해되는가"**

Sanctum OS 정체성:
- "걸어다니는 성경" — 묵상·기도가 모든 회복의 종착지
- 원칙·판례·목표·도트·리포트가 **한 흐름의 인과 체인** (메모리 [feedback_organism_architecture.md](../../.claude/projects/c--Users-MSI-Desktop----Sanctum-OS/memory/feedback_organism_architecture.md))

**1차 베타의 본질**: 베타 테스터가 이 시스템을 사용한 후 **"이거 다 연결돼 있구나"** 가 보이는가, 아니면 **"이거 기도 일기 앱 / 도트 트래커 / 평가 도구 따로따로네"** 로 보이는가의 분기.

→ 1차는 **정성** 1순위, **정량** 보조.
→ 정량 보조 신호: GA4 모듈간 이동률 (도트 → 원칙 → 목표 / 분별의 자리 → 폐기 게이트 / 묵상 → 기도 → 리포트), 인앱 피드백 누적량.

**2차 베타의 본질**: 공동체 기능이 같은 흐름에 자연스럽게 합류하는가. 1차의 개인 흐름이 관계 신뢰도·양치기 소년·기여도로 확장될 때도 "한 유기체" 감각이 유지되는가.

## §2. 1차 클로즈 베타

### §2.1 시작 인원·확장 결 (v6 — 2026-05-19 갱신)

**v6 결로 명단 자체 관리 폐기**. v6 자리 = 시작 14명 + 추천 링크 결로 자연 확장 → 100명 도달 시점에 2차 베타 진입.

**핵심**:
- **시작 인원**: 14명 (1:1 권유 응답 누적)
- **확장 방식**: 추천 링크 결로 *14명이 다른 사람 초대* → 자연 확장
- **종료 시점**: 100명 도달 = 2차 베타 시작 시점
- **명단 관리 폐기 이유**: 사용자 명시 *"쟤들이 결국에는 사람 초대하면 이제 누가 누군지는 별로 안 중요함"*. v6 결 정합 — 자연 확장에서 명단 표·역할 분류 의미 X.
- **이전 v5 9명 명단 표**: 폐기. 신앙 연차 보강 가설도 함께 폐기 (*"오래 믿었다고 다 믿음 좋은 거 아님"*).

**모집 방식**: 1차 직접 권유 14명 + 자발 추천. 공개 모집 X (반-오픈 결).

**비신자**: 1차 타겟 아니나 추천 결로 자연 진입 가능. 동의 절차는 같음.

**인터뷰 부담**: 30분 × N명. 인원 100명 도달까지 자연 누적. 핵심 결로 *14일 미션 완수자* 우선 인터뷰.

### §2.2-D 온보딩 흐름 v1 — 2026-05-14

가입 직후 9~20명이 처음 마주칠 화면들의 순서·텍스트. 보안 재기획 미진입 상태(현재 시스템 그대로)에서 작동하는 *최소 친절* 버전.

**전체 순서**:

```
1. 가입 폼 (이메일·마스터 비밀번호)
2. 24단어 생성·안내 모달  ←  §2.2-D ①
3. 동의 모달 (§2.2-C)
4. 첫 화면 = 본인 프로필 Day 0 (오리엔테이션)  ←  §2.2-D ②
5. (사용자가 막히면) 1:1 카톡 셋업  ←  §2.2-D ③
```

**순서 결정 (§2.2-C 미해결 갈래 ① 답)**: **24단어 먼저 → 동의 모달**. 이유 = 동의 모달 본문이 *"24단어로 잠겨요"*를 전제하므로 안내 먼저가 자연스럽다.

## §3. 1차 → 2차 게이트

### §3.1 게이트 정의

> **1차 리더·전문가 전원의 "내 사람에게 권하겠음" 명시**
