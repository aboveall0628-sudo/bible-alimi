# Sanctum OS — 사업 운영 로드맵 v1 (7년 풍년·흉년)

**작성**: 2026-05-15
**용도**: 1인 개발자의 신앙 기반 앱 사업 운영 좌표계. 베타 → 출시 → 캐시카우 → 확장 → 흉년 대비까지 7년 그림. 보안 재기획 기획서·베타테스트 기획서와 함께 사업 의사결정의 3대 기준 문서.

## 🙏 0. 모든 운영의 0원칙

> **말씀으로 방향 점검. 선택 전 하나님께 묻고 응답 기다리기. 평안 오는 방향으로 움직이기.**

어떤 결정도 이 원칙 위에서 검토해요. 0원칙이 깨지면 아래 모든 KPI·로드맵·리스크 매트릭스가 다 무력해져요.

### 운영 형태

| 시점 | 멈춤 → 묵상 → 기도 → 평안 확인 |
|---|---|
| 일일 | 그날 의사결정 1개(가장 무거운 것)는 묵상 안에 통과시키기 |
| 주간 | 주 1회 "이번 주 결정이 평안 안에 있었나" 회고 |
| 월간 | 매출·사용자 데이터 보기 전에 먼저 말씀과 기도 |
| 분기 | "지금 가고 있는 방향이 부르심에 맞는가" 깊게 점검 |
| 매출 마일스톤 | 도달 시 자기 점검 — "돈이 들어오면서 내가 변하지 않았는가" |

### 0원칙 위반 신호 (이 중 하나라도 발생하면 즉시 멈춤)

- 결정이 급해서 기도가 사라졌어요
- "이거 안 하면 망한다"는 압박이 평안보다 커요
- 묵상이 사업 분석으로 변질됐어요
- 주일·안식 시간이 일에 침범당했어요
- 가족·공동체와의 약속이 일에 밀려요

이 5개 신호가 누적 2주 이상이면 분기 점검을 앞당겨 진행해요.

### 시스템 자리 연결
- 의사결정 보조 AI(B-1)의 **원칙 1번**으로 등록 검토 — 모든 게이트가 이 원칙을 거쳐 작동
- 외부 영적 멘토 1명 분기 1회 점검 (Phase 0부터 시작)
- 사역 vs 사업 정체성 점검은 분기 회고에 박지 않고 자연스럽게 자리잡기

## Phase 0 — 검증기 (현재 ~ 6개월)

### 목표
- 1차 클로즈드 베타(10명·2주) → 2차 오픈 베타(100명·4주) → 매트릭스 판정
- 개인사업자 등록 (2차 베타 직후)
- HC 정비 + CS AI 운영 자동화 1차

### 사용자 & 매출
| 시나리오 | 사용자 (6개월 차) | 월 매출 |
|---|---|---|
| 보수 | 100~300명 | 0 ~ 25만 원 |
| 낙관 | 500~1,000명 | 25 ~ 50만 원 |

### 수익 배분
- 매출 미미 → 적립 의무 없음
- 운영비·검증 비용 100% (누적 30만 원 미만 한도 유지)
- **풍년 적립률 0%**

### 주요 액션
- **제품**: 베타 매트릭스 A·B·C 분기. HC 정비 Phase 1~4
- **법무·세무**: 정식 출시 직전 개인사업자 등록 (간이과세자로 시작). 업종 코드 722000(시스템·응용 소프트웨어 개발·공급업) + 부업 코드(콘텐츠·교육) 추가 검토. 이용약관·개인정보처리방침 초안에 변호사 1회 자문 (5~20만 원 예산)
- **마케팅**: 0원. 1:1 권유만
- **콘텐츠 사역**: 미진입 (자원 분산 금지)
- **해외**: 미진입

### 리스크 & 대응
- PMF 부재 → 30만 원 미만 학습 비용으로 한정. 매트릭스 G면 중단 검토
- 종교 민감 데이터 사고 → 묵상·기도 별도 컬렉션 + 암호화 분리(현 구조) 점검
- 개발자 번아웃 → Sabbath 토글 + 1일 작업 상한 자기 룰

### 다음 Phase 진입 트리거
- 베타 매트릭스 A(Go) 또는 B(부분 피벗, 묵상→실행 통함)
- "사라지면 아쉽다" 평균 ≥ 7점
- D14 리텐션 ≥ 50%

### 축소·중단 트리거
- 매트릭스 G + 증거 1 Weak
- 0원칙 점검 시 "이 방향이 부르심이 아니다" 평안

- 묵상 데이터 침해 사고 → 보안 재기획 트랙 본격 진입

  - **팟캐스트**: 묵상 운영기, 신앙·일·돈 다루는 토크. 월 4회 정기. 광고·후원 BM

### 리스크 & 대응
- 영어 진입 실패 → 한국·재외 한인 시장 안 1등 굳히기. 글로벌 야망 1년 보류

- 데이터 침해 사고 (GDPR/CCPA) → 보안 재기획 트랙 완료 필수

| 세금 예비 | 5% (2.5만) | 10% (10만) | 10% (50만) | 10% (100만) | 10% (250만) |

핵심 원리:
- 매출 구간이 올라갈수록 생활비 비중은 줄어요 (고정비 성격)
- 풍년 기금은 매출 증가에 비례해서 적극 늘려요
- 사역·콘텐츠는 매출 500만 이후 본격 활성, 2,500만 이후 10%로 단계 상향
- 세금 예비는 일반과세자 진입 후 10% 유지가 안전선

### 흉년 기금 목표 적립액 산출 공식

```
목표 적립액 = 월 고정비 × 보유 희망 개월 수
```

월 고정비 구성: 본인 생활비 + 사업 운영 고정비 + 인건비 + 부동산

보유 희망 개월 수 권장:
- **Phase 0~1**: 3~6개월 (검증기, 빠른 사이클)
- **Phase 2**: 6~12개월 (캐시카우 안정 자리)
- **Phase 3**: 12~18개월 (실험기, 실패 비용 대비)
- **Phase 4**: 12~18개월 + 사역 펀드 별도

예시 계산:
| 시점 | 월 고정비 가정 | 목표 개월 수 | 목표 적립액 |
|---|---|---|---|
| Phase 1 후반 | 200만 원 | 6개월 | 1,200만 원 |
| Phase 2 중반 | 500만 원 | 9개월 | 4,500만 원 |
| Phase 3 중반 | 1,500만 원 | 12개월 | 1.8억 원 |
| Phase 4 안정 | 3,000만 원 | 15개월 | 4.5억 원 |

### 기금 운용 원칙

#### 유동성 확보 방법 (안전 위주)
- **CMA**: 일 단위 이자, 즉시 인출. 적립 기금의 50~70%
- **MMF**: 단기 채권 중심, 안정성 높음. 20~30%
- **정기예금 (분할 만기)**: 3·6·12개월 분할 예치. 10~20%
- 외화 비중: 매출의 30% 이상이 외화면 달러 CMA·MMF 분산

#### 절대 투자하지 않을 항목
- 주식 직접 투자 (변동성)
- 암호화폐
- P2P 대출
- 부동산 (자가 사무실 제외)
- 친지·지인 사업 투자

#### 분리 원칙
- 흉년 기금 계좌는 **사업·생활 계좌와 물리적으로 분리**
- 매월 자동 이체로 적립 (의지 의존 X)
- 인출 시 본인 + 외부 1명(가족·멘토) 동의 룰 (자기 견제)

### 위기 시 기금 인출 트리거

| 트리거 | 인출 한도 | 추가 조건 |
|---|---|---|
| 매출 3개월 연속 30% 이상 하락 | 운영비 부족분 × 6개월치 한도 | 0원칙 점검 + 멘토 1명 의견 |
| 예상치 못한 법적 비용 (소송·민원·앱스토어 분쟁) | 실제 비용 + 예비 20% | 변호사 견적서 첨부 |
| 데이터 침해 사고 대응 | 사고 추정 비용 × 1.5배 | 사고 보고서 + 외부 보안 자문 |
| 개발자 본인 건강·가족 위기 | 6개월 휴식 가능 수준 | 0원칙 점검 — 우선 멈춤 |
| 전략적 기회 (인수·합병·핵심 인재 영입) | 흉년 기금의 30% 이내 | 분기 점검 통과 + 0원칙 평안 |

인출 후 의무:
- 인출 즉시 재적립 계획 수립 (몇 개월 안에 복구할지)
- 다음 분기 매출의 풍년 적립률을 +10%p 상향해 빠른 복구
- 인출 사유 기록 → 의사결정 보조 AI의 판례로 적재
